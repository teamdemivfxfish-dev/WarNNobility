Antique Atlas Map – Test Checklist

getting War ’n Nobility’s domain borders to draw on the map
War ’n Nobility • NeoForge 1.21.1 • tick each box once it is confirmed working

Work top to bottom. The mods must be right before the world data, and the world data must exist before any borders can draw. If a later step fails, the cause is almost always a missed box higher up.

Part 1  The mods are all present and the game boots

Antique Atlas 4 and its Fabric bridge are installed
  • Antique Atlas 4 is a Fabric mod, so on NeoForge it rides on Sinytra Connector and needs the whole bridge stack:
  • antique-atlas (the map itself)
  • connector (Sinytra Connector)
  • forgified-fabric-api
  • surveyor (Antique Atlas’s map-data backend)
MineColonies is installed
  • The domain map only runs when MineColonies is present; the whole overlay is gated behind it.
  • Without MineColonies the rest of War ’n Nobility still works, but no borders will ever draw.
War ’n Nobility is installed (the build with the Domain Atlas merged in)
  • Use the NeoForge jar (warnnobility-neoforge-1.0.0.jar); the domain map lives only in the NeoForge build.
Easy Factions is installed (only needed for the Factions layer)
  • Required for the Factions overlay to appear and draw faction borders.
  • The Colonies / Counties / Duchies / Kingdoms layers do not need it.
The instance launches all the way to the world with no errors
  • Boot the pack once and reach the title screen, then load into a world.
  • Watch for a “mod missing dependency” crash on the loading screen; if it appears, a mod from Part 1 is absent.
Heads-up: Sinytra Connector inside a large NeoForge pack can occasionally clash with another mod. If the game crashes only after adding the Atlas stack, that bridge is the first suspect, read the crash log for “connector” or “fabric”.

Part 2  The world has something worth drawing

At least one MineColonies colony exists with claimed chunks
  • Place a Town Hall and let the colony claim its surrounding chunks, this is the raw land the borders are built from.
You are a Count or higher with a named domain
  • Borders aggregate up the vassal tree: barons’ land sums into a County, counties into a Duchy, duchies into a Kingdom.
  • A plain Baron with no vassals has no county border yet, climb to Count+ first.
  • Name your land at the Chancery Table › Court tab, or with /nobility domain <you> <name>; the name becomes the label on the map.
(For the Factions layer) you lead an Easy Factions faction with claimed chunks
  • Found a faction and claim some chunks so the Factions overlay has territory to colour.

Part 3  The map draws the borders

Open the Antique Atlas map
  • Obtain the Antique Atlas item, then open it with the Antique Atlas keybind (or by right-clicking the atlas).
The domain toggle buttons appear on the map
  • Look to the side of the open map for the toggle row.
  • You should see five toggles: Colonies, Counties, Duchies, Kingdoms, Factions.
  • If the toggles are missing entirely, the overlay did not hook in, recheck MineColonies + Antique Atlas in Part 1.
Each overlay actually draws when toggled on
  • Colonies → the raw MineColonies claims fill in.
  • Counties / Duchies / Kingdoms → coloured pencil-edge borders with a name label at the heart of each realm.
  • Factions → each faction tinted its own colour and labelled.
Borders refresh on demand
  • If a border looks stale or missing after you change colonies/ranks, run /domains refresh and reopen the map.
The drawn borders match your setup
  • A kingdom’s border should equal the sum of all the land its vassals hold.
  • The labels on the map should read the domain names you set in Part 2.
